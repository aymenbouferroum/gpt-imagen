# Prompt Patterns

## General image spec

```text
Use case: <mockup | concept art | figure | product shot | edit>
Asset type: <hero image | sprite sheet | paper figure | ad creative>
Primary request: <main ask>
Input images: <roles for references or edit targets>
Scene/backdrop: <environment>
Subject: <main visual subject>
Style/medium: <photo | 3D render | editorial | infographic | pixel art>
Composition/framing: <camera angle, crop, layout>
Lighting/mood: <tone and lighting>
Color palette: <palette constraints>
Constraints: <must preserve or must include>
Avoid: <negative constraints>
```

## Frontend mockup

- ask for device framing or browser framing only if it adds value
- specify negative space for headline copy when relevant
- avoid impossible UI density
- avoid purple-on-black unless requested

## Game assets

- specify whether the output is icon, prop, character sheet, key art, or environment concept
- mention transparent background when the asset will be composited later
- specify consistent style family when requesting a set

## Paper visuals

- use this path for conceptual figures, not measured-data charts
- describe the system or mechanism clearly
- ask for clean academic visual hierarchy instead of "make it fancy"
